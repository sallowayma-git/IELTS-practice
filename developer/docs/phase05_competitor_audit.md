# Phase 05 竞品公开行为审计

> 文档更新时间: 2026-04-07
> 审计范围: `https://www.ielts9.me/`、`https://ieltsrewind.com/zh`
> 审计方法: 仅基于公开页面、公开文案、公开网络行为与 Playwright 实测；不绕过认证，不碰私有接口，不抄源码。

---

## 摘要

Based on current information, my understanding is:
这轮审计要解决的不是“抄谁的 UI”，而是搞清楚两个公开产品分别靠什么完成转化、评分呈现和能力分层。结论很清楚：`ielts9.me` 赢在结果页产品化和门槛设计，`IELTS Rewind` 赢在免费工具矩阵和低摩擦入口。我们当前短板主要是产品编排层，不是单次评分 API 不够多。

---

## 【核心判断】

✅ 值得迁移，但只迁移公开可观察到的产品结构和业务逻辑，不迁移脆弱实现细节。

真正该学的是：

- `ielts9.me` 的“首页即结果页 demo + 高价值能力 gated”
- `IELTS Rewind` 的“免费矩阵入口 + 写作/口语/资料整合”

真正不该学的是：

- 把所有结果都塞进一次大 prompt
- 把核心能力绑死在第三方会话桥接上
- 为兼容历史脏数据继续堆补丁分支

---

## 【关键洞察】

### 1. `ielts9.me`：结果页产品化远强于单次评分能力

公开可见能力：

- 首页直接展示 demo 批改结果，不是空白输入页
- `Task 1 / Task 2`
- `剑雅真题 / 当季新题 / 随机题目 / 生成观点`
- 题目与作文双输入，均有 `图片转文字`
- `连词助手 / 生成范文 / 开始计时`
- 结果区分层：
  - `批改`
  - `近义词`
  - `高分词`
  - `话题词伙`
- 句级点评卡片：
  - `Good Point`
  - `Suggestion`
  - `点击翻译`
- `Overall Band Score` + 四维评分 + 长段解释

门槛设计：

- 未登录点击 `写作复盘/提分报告` -> 登录弹窗
- 未登录点击 `批改记录` -> 登录弹窗
- 未登录点击 `草稿箱` -> 登录弹窗
- FAQ 公开可读，不 gated

公开业务规则：

- FAQ 明示 `复盘报告`：`69.99 / 129.99` 套餐免费，否则 `1 积分 / 次`
- FAQ 明示 `范文生成`：套餐免费，否则 `1 积分 / 次`
- FAQ 明示 `同一篇作文仅能批改 1 次`
- FAQ 明示 `批改记录` 可下载 `word/pdf`
- FAQ 明示积分无使用期限

公开技术迹象：

- `GET /api/auth-status -> 401`
- 存在 `clerk.ielts9.me` 与 `accounts.ielts9.me/sign-in`
- `personal-center` 存在跳转

推导：

- 它的增长漏斗不是“先登录再试”，而是“先看到结果价值，再为连续性能力付费/注册”
- 它的核心资产不是一次评分，而是：
  - 结果页可感知价值
  - 学习层
  - 连续性资产（历史、草稿、复盘、导出）

### 2. `IELTS Rewind`：免费工具矩阵很强，但写作闭环深度较浅

首页和工具矩阵公开能力：

- `essay-checker`
- `report-checker`
- `letter-checker`
- `writing-essays`
- `writing-reports`
- `speaking-practice`
- cue cards / introductions / reading / listening

`essay-checker` 公开页面能力：

- 单页双输入：题目 + 作文
- 题目与作文均支持：
  - `输入文本`
  - `上传图片`
- 静态说明页明确宣称按四大 rubric 分析
- 强调：
  - 免费
  - 即时反馈
  - 语法/词汇/结构分析
  - 无需注册

Playwright 实测提交结果：

- 在 2026-04-07 的无会话环境下，输入真实题目与 257 词作文后点击 `批改我的作文`
- 页面进入 `正在分析您的作文...`
- 同时新开 `Puter` 标签页：`https://puter.com/?embedded_in_popup=true&request_auth=true`
- 网络可见：
  - `api.puter.com/whoami -> 401`
  - `socket.io` polling
- 等待后未拿到公开结果页，主页面持续停留在分析中

推导：

- 它的公开入口摩擦确实低，但评分执行依赖第三方会话/授权桥
- 至少在干净环境下，“无需注册”并不等于“无任何外部授权依赖”
- 它更像免费流量入口与资料站，而不是深度学习闭环产品

### 3. 对照我们当前仓库，真正缺的是产品层，不是再堆 prompt

我们已有：

- `scoring -> review` 双阶段评分链路
- `Compose -> Evaluating -> Result -> History` 基础闭环
- 自由写作 / 题库模式 / 草稿恢复 / 历史记录 / 导出基础

我们明显缺：

- 首页级 demo 结果体验
- 结果页学习层：
  - 近义词
  - 高分词
  - 话题词伙
  - 单条翻译
- 写作辅助层：
  - 生成观点
  - 连词助手
  - 计时器
  - OCR
- 高价值连续性能力的明确分层：
  - 复盘报告
  - 批改记录中心
  - 草稿箱
  - 导出产品化

### 4. 当前实现中最该砍掉的垃圾复杂度

- 结果协议前后端存在多套兼容分支，说明契约没压实
- 继续把分析、句评、词汇、复盘塞进单轮 prompt，只会放大波动
- 如果未来也学 `IELTS Rewind` 走外部桥接授权，主链路会变脆

## 【迁移优先级】

### P0：先迁移结果页结构，不先扩更多模型调用

目标：

- 首页或写作入口提供可交互 demo 结果体验
- 结果页拆成三个消费层

建议结构：

1. `scorecard` 层
   - 总分
   - 四维分
   - 核心诊断标签
2. `review` 层
   - 句级亮点
   - 句级问题
   - 段落诊断
3. `vocabulary` 层
   - 近义词
   - 高分词
   - 话题词伙
   - 可选翻译

实现原则：

- 第一阶段只产出严格 JSON 骨架，直接驱动评分仪表盘
- 第二阶段并发产出句级评审与词汇层
- 第三阶段再按需生成复盘报告或范文

### P1：补写作辅助层

- 生成观点
- 连词助手
- 计时器
- OCR 输入链路

注意：

- 这些功能都该围绕写作任务服务
- 不要做成彼此孤立的按钮坟场

### P1：补连续性资产与门槛设计

- `批改记录`
- `草稿箱`
- `复盘报告`
- `Word/PDF` 导出

策略：

- 首次评分尽量开放
- 连续性与高价值总结能力 gated
- 门槛公开透明，不搞暗规则

### P2：补业务规则文档化

- FAQ
- 积分/额度说明
- 报告生成规则
- 历史与导出边界

| 能力 | `ielts9.me` | `IELTS Rewind` | 我们当前 | 建议 |
| --- | --- | --- | --- | --- |
| 首页 demo 结果页 | 有 | 无 | 无 | P0 迁移 |
| 免费首用输入评分 | 有 | 有 | 有 | 保留并强化 |
| 题库入口 | 有 | 有 | 有 | 强化筛选与元数据 |
| 生成观点 | 有 | 未见 | 无 | P1 迁移 |
| 连词助手 | 有 | 未见 | 无 | P1 迁移 |
| OCR | 有 | 有 | 无完整链路 | P1 迁移 |
| 句级点评 | 有 | 提交结果未验证 | 有基础 | 继续压质量 |
| 词汇学习层 | 有 | 未见 | 无 | P0-P1 迁移 |

---

## 【Linus式方案】

1. 先压数据契约
   - 定义单一 `EvaluationContract v3`
   - 分开 `scorecard / review / vocabulary / report`

2. 再压产品链路
   - 首页 demo
   - 结果页分层
   - 连续性能力 gated

3. 最后再补工具层
   - OCR
   - 观点
   - 连词
   - 计时

4. 不做这些垃圾事
   - 不继续堆单轮超级 prompt
   - 不为兼容旧垃圾结构无限打补丁
   - 不把关键能力绑到不可控第三方桥接

---

## 结论

`ielts9.me` 证明了：写作评分产品真正值钱的是“结果页产品化 + 学习层 + 连续性资产”，不是一次打多少 token。`IELTS Rewind` 证明了：免费矩阵入口有流量价值，但如果评分执行依赖外部桥接，会让主链路变脆。我们下一步应该优先做结构化多阶段输出和结果页分层，而不是继续往一个 prompt 里塞更多花活。
