# 阶段四任务书：功能完善与体验优化

## 版本信息
- 项目：IELTS Practice (IELTS Atlas)
- 当前版本：0.3.1
- 分支：opensource
- 阶段目标版本：0.4.0

## 阶段背景

阶段一至阶段五的重构工作已全部完成（紧急修复、架构重构、数据层优化、性能治理、代码质量改进）。阶段六（测试覆盖与CI）持续推进中。当前系统核心功能稳定，阅读练习、听力练习、套题模式、练习记录、词汇辅助、成就系统、数据管理等模块均已上线。

本阶段（阶段四·功能完善）聚焦于：补齐已规划但未实现的功能模块、提升移动端体验、增强数据洞察与学习激励、扩展测试覆盖、优化构建产物。

## 核心目标

1. **学习目标系统**：实现 GoalManager UI 集成，支持日/周/月学习目标设定与追踪
2. **移动端体验优化**：响应式布局完善，触控交互适配
3. **数据洞察增强**：总览仪表盘增加学习趋势、薄弱环节分析等卡片
4. **测试覆盖扩展**：新增核心模块单元测试与E2E回归测试
5. **构建与性能优化**：Bundle 分析与瘦身，懒加载策略优化

## 技术约束

1. **file:// 兼容**：所有功能必须在 file:// 协议下正常运行，不依赖后端服务
2. **零破坏**：不破坏现有阅读、听力、套题、记录、词汇等流程
3. **构建验证**：每次功能改动后运行 `node scripts/build-bundles.mjs` 重建 bundle
4. **测试验证**：功能改动后运行静态分析和 E2E 测试
5. **发布包兼容**：新增文件需在 developer/release.sh 排除规则中正确处理
6. **Linus 哲学**：简化数据结构、消除特殊情况、保证零回归
